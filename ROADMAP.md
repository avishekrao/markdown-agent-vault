# Roadmap

This roadmap keeps the starter pack focused on making file-based AI work understandable, safe, and repeatable.

## v0.1.0 - First Public Starter Pack

Status: released.

Goals:

- public README explains the method in 30-120 seconds;
- quickstart gives a safe first test;
- license and contribution files are present;
- one minimal example shows the first-session workflow;
- local validation scripts run without external services.

## v0.2.0 - Project Creation, Memory, and Contour Repositories

Status: released.

Goals:

- add `project-creator` for complete agent-created projects;
- add autonomous planning guidance with Intent Lock, Owner Interaction Policy, and Quality Criteria;
- separate current memory from archive sources;
- add memory trust fields, ledger, anti-memory, conflicts, and audit templates;
- add compressed meeting history through `context-compression` and `meetings/README.md`;
- document two usage modes: personal local vault and GitHub contour repository;
- add `repository-manifest.yml` for machine-readable repository boundaries;
- add templates for contour `AGENTS.md`, `README.md`, and change-request text;
- add a safe minimal contour repository example;
- add local validation for contour repositories.

## Later

Possible:

- collect first-run feedback from early users;
- add more realistic public examples;
- add a checklist for adapting the starter pack to an existing Markdown vault;
- document known limitations;
- add tested setup notes for specific file-aware agents;
- add a compatibility report template;
- add stronger validation scripts;
- add a smaller distribution profile for people who only want the core method.

## Non-Goals

- becoming a hosted memory product;
- replacing Obsidian, GitHub Issues, or project trackers;
- promising universal AI memory;
- adding automation before the manual method is clear.

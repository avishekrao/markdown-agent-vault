# Release v0.2.0

Project creation, vault memory, meeting compression, and GitHub contour repositories.

## Added

- `project-creator`: a skill that creates a full project file set instead of asking the owner to write the plan.
- Stronger project planning: intent lock, owner interaction policy, quality criteria, blockers, drift guard, and review protocol.
- `vault-memory`: current memory is separated from archive, logs, old meetings, drafts, and imports.
- Memory trust layer: memory ledger, anti-memory, conflicts, and templates for memory cards, conflicts, audits, and working context.
- `context-compression`: compressed `meetings/README.md` for long meeting histories.
- GitHub contour repository mode: `repository-manifest.yml`, templates, validation, and an example repository.
- Validation scripts for repository manifests and contour repositories.
- Agent intent reconstruction methodology for underspecified requests.

## Changed

- `AGENTS.md`, onboarding, quickstart, write protocol, task routing, and project templates now point agents toward explicit intent, current memory, and safer project creation.
- Meeting processing and project workflows now avoid treating archives and old notes as current truth.
- Public examples and release notes describe how to use the starter pack for longer real work, not only the first session.

## Why This Matters

The starter pack now supports real long-running work:

- projects start faster and more consistently;
- important memory claims are easier to verify;
- meeting history can be compressed without losing decision chains;
- a shared contour can be moved into GitHub with boundaries, validation, and human review.

# Context

## Essence

Stable context for the example contour repository.

## Current Invariants

- This is a public-safe example.
- It demonstrates repository boundaries, manifest-driven agent rules, and validation.
- The repository is not a real business contour and does not store private material.

## Source of Truth

- `repository-manifest.yml` defines repository policy.
- `AGENTS.md` defines agent behavior.
- `README.md` provides human navigation.

## Next step

Use this example as a template for real contour repositories after adapting names, owners, access rules, and validation.

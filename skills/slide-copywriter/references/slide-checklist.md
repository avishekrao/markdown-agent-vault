# Slides content quality checklist

Go through each point before giving the result to the user.

## Deck level (macrostructure)

- [ ] SCQA is formulated and agreed with the user
- [ ] Answer (Answer) on slide 1-2, not at the end
- [ ] Title-only test: Read all the headlines in a row – does it make a coherent story?
- [ ] There is an alternation of what is what can be (Duarte Sparkline)
- [ ] Each slide leads to a conclusion or a decision (no information slides without action)
- [ ] Total volume: 15-25 slides for live presentation
- [ ] If there is more data than fits in, there is a handout / appendix

## Slide level (microstructure)

- [ ] Title = action title, not topic description
- [ ] Title ≤12 words
- [ ] Body: ≤6 visual elements (Rule of Six)
- [ ] One thought on a slide (exceptions: comparison, scorecard)
- [ ] There are no full paragraphs on the slide – only key phrases, numbers, graphs
- [ ] Specified type of visualization (table / graph / diagram / bullet points)
- [ ] Speaker notes: 3-5 sentences (context + transition + anticipation of questions)

## Text clarity

- [ ] No introductory designs ("should be noted", "important to emphasize")
- [ ] There are no imposed assessments without facts ("unique", "effective", "innovative")
- [ ] No office (“to carry out”, “for purposes”, “within”, “based”)
- [ ] No uncertainty ("some", "row", "different") - replaced by numbers
- [ ] A valid pledge instead of a suffering one
- [ ] Bullet points according to the formula: [What] + [How much] + [So what]
- [ ] After garbage removal, the text is filled with facts (not dry)

## Data slides

- [ ] Table ≤6 lines (the rest in handout)
- [ ] There are RAG indicators / color coding for quick readability
- [ ] Removed grids, 3D effects, decorative elements
- [ ] Inference from data is in the header, data is like proof in the body

## Final inspection

- [ ] No antipatterns: descriptive headlines, detective intrigue, copy-paste tables, slides about everything
- [ ] Each slide answers the question, "So what?"
- [ ] If it's a stratasession, each slide leads to a fork or recommendation.
